
#  Uso de Markdown en proyectos de software
##  ¿Qué es Markdown y por qué se utiliza?
**Markdown** es un lenguaje de marcado ligero creado por *John Gruber (2004)* con el propósito de simplificar la redacción de documentos estructurados. Su sintaxis es sencilla, legible y permite transformar el texto en formatos como **HTML, PDF o DOCX**, lo que lo hace ideal para proyectos de software y documentación técnica.
Según **Giancarlo Fontela da Luz** en su obra *“Markdown intrínsecamente fascinante: conversión de archivos Markdown a PDF mediante tecnologías web”*, el uso de Markdown ha demostrado ser una herramienta poderosa en la **documentación de proyectos de software**, ya que permite a los equipos:
- Crear documentación rápida y clara.  
- Personalizar estilos mediante **HTML, CSS y JavaScript**.  
- Convertir archivos Markdown a PDF de forma automatizada.  
- Mantener los documentos junto al código fuente en repositorios como GitHub o NPM.
El producto desarrollado por Fontela da Luz siguió el **modelo en cascada**, con un diseño bien estructurado y el uso de **diagramas UML** para visualizar componentes. Además, implementó **pruebas automatizadas con un 80 % de cobertura**, asegurando la calidad del código y la documentación.

##  Ventajas de usar Markdown en combinación con GitHub
1. **Integración directa:** GitHub interpreta automáticamente los archivos `.md`, mostrando la documentación con formato legible.  
2. **Versionado controlado:** Cada modificación queda registrada mediante commits, lo que permite restaurar versiones anteriores.  
3. **Colaboración efectiva:** Varios desarrolladores pueden editar la documentación de forma simultánea.  
4. **Conversión multiplataforma:** Markdown se puede exportar fácilmente a PDF, HTML o Word sin perder formato.  
5. **Profesionalismo y claridad:** Facilita la lectura técnica y estandariza la presentación de los documentos del proyecto.

##  Aplicación práctica
En este proyecto de software, Markdown se utilizó para:
- Crear el archivo **README.md** con la descripción general del sistema.  
- Redactar la propuesta de mantenimiento y documentación técnica.  
- Integrar la información del proyecto con GitHub para control de versiones.  
- Permitir que cualquier usuario consulte la documentación sin herramientas adicionales.
